What Was Delivered

This document covers everything you need to operate the Jifs Company system — from day-to-day use to managing backups and moving the system to a new computer.

What Is This System?

A custom-built operations management system for Jifs Company & Gen Supp Ltd that runs entirely on your Mac computer — no internet required. It tracks vehicles, suppliers, expenses, services, parts, and generates financial statements and reports.

backup-jifs.command

Double-click to save a timestamped copy of all data

reset-jifs.command

Wipes ALL data for a completely fresh start

All system files — do not delete or rename this folder

This document — open in browser and print to PDF

Chapter 1
Starting & Stopping the System

Starting the System

  1. 1
    Go to the Desktop

    Find the file called start-jifs.command

  2. 2
    Double-click it

    A black terminal window will open briefly — this is normal. It is starting the system.

  3. 3
    Wait for the browser

    Safari or Chrome opens automatically at http://localhost:3000

  4. 4
    Notification appears

    Shows the local network address so other computers on the same WiFi can also access the system.

First Time Only

The very first startup takes 1–3 minutes to install software. Every startup after that is fast.

Stopping the System

Double-click stop-jifs.command on the Desktop. Always stop before shutting down the Mac.

Important

Do not shut down the Mac without stopping the system first. Always stop it to avoid any risk of data corruption.

Accessing from Another Computer on the Same WiFi

Any computer or phone on the same WiFi can open a browser and go to the LAN address shown in the startup notification — for example: http://192.168.1.5:3000

Note

The LAN address may change if the Mac restarts. The startup notification always shows the current address.

Chapter 2
First-Time Login & Admin Setup

The first time the system starts with a fresh database, you must create the main Admin account. This happens only once.

Setting Up the Admin Account

  1. 1
    Open the system

    The login page shows a "First-Time Setup" screen instead of the normal login.

  2. 2
    Choose a username

    Something simple like your name or admin. No spaces.

  3. 3
    Set a password

    At least 8 characters including one number. Example: Jifs2026

  4. 4
    Confirm the password

    Type it again exactly.

  5. 5
    Click "Create Admin Account"

    You will be taken to the normal login screen to sign in.

⚠ Write This Down Immediately

Write your username and password on paper and keep it somewhere safe. If you forget the Admin password there is no way to recover it — a developer will need to reset the entire database.

Normal Login

Enter your username and password on the login screen and click Sign In.

Logging Out

Click the Sign Out button at the bottom of the left sidebar.

  2. 2

Chapter 12
Backup & Restore

All your data lives in one file: fleet.db. Backing up means copying this file safely. Do this regularly.

Making a Backup

  1. 1
    Double-click backup-jifs.command

    Found on the Desktop.

  2. 2
    Done automatically

    A timestamped copy is saved to: Desktop → JIFS-Backups → fleet_2026-04-23_14-30.db

  3. 3
    Copy to a USB drive

    Plug in a USB drive and copy the entire JIFS-Backups folder to it for safekeeping.

Recommended Schedule

Run backup every Friday before closing. Copy to USB once a month. The script keeps the last 30 backups automatically.

Restoring from a Backup

  1. 1
    Stop the system

    Double-click stop-jifs.command

  2. 2
    Open the data folder

    Navigate to jifs-fleet → backend → data

  3. 3
    Replace the database

    Delete the current fleet.db, then copy your backup file here and rename it exactly to fleet.db

  4. 4
    Restart

    Double-click start-jifs.command. All restored data will be there.

Warning

Restoring replaces ALL current data with the backup. Any entries after the backup date will be lost. Only restore when necessary.

Chapter 13
Moving System to a New Mac

To move the system to a different Mac, follow these exact steps in order.

Step 1 — Install Node.js on the New Mac

Open a browser on the new Mac and go to nodejs.org. Download the LTS version and install it. To confirm, open Terminal and type:

node -v

It should print a version number like v22.0.0.

Step 2 — Copy the Project Folder

Copy the entire jifs-fleet folder from the old Mac to the new Mac’s Desktop. Use a USB drive, AirDrop, or external hard drive.

Make the Transfer Faster

Before copying, you can delete the backend/node_modules folder inside jifs-fleet. It is large and will be reinstalled automatically. Do not delete anything else.

Step 3 — Make the Scripts Executable

Open Terminal on the new Mac and run:

chmod +x ~/Desktop/jifs-fleet/*.command

If you placed the folder somewhere other than the Desktop, adjust the path.

Step 4 — Start the System

Double-click start-jifs.command. The first run automatically installs dependencies and rebuilds the database driver for the new Mac’s version of Node.js — this prevents the "module version mismatch" error that used to happen.

Step 5 — Fresh Start or Keep Existing Data?

SituationWhat to Do
You want to keep all existing data from the old MacJust log in normally — all data will be there
You want a completely fresh start with no old dataRun reset-jifs.command first, then restart — see Chapter 14
Note on First-Time Setup Screen

If the setup screen is not showing on the new Mac after copying, it means the old database was included in the copy. Run reset-jifs.command to wipe it, then restart to get the setup screen.

Chapter 14
Resetting to Factory Fresh

Resetting wipes ALL data — vehicles, suppliers, expenses, users, everything — and returns the system to its original state. This cannot be undone.

⚠ Warning — This Deletes Everything

Make a backup using backup-jifs.command first if you want to keep a copy. A reset is permanent.

How to Reset

  1. 1
    Double-click reset-jifs.command

    Found on the Desktop.

  2. 2
    Confirm the dialog

    Click "Yes, Reset Everything" to proceed, or Cancel to abort.

  3. 3
    Wait for the notification

    A notification confirms the reset is complete.

  4. 4
    Restart the system

    Double-click start-jifs.command. The First-Time Setup screen will appear.

When Would You Reset?

  • Moving the system to a new computer and starting fresh
  • Clearing test data before real business use begins
  • Starting over after a major restructuring

Browser opens but shows "This site can't be reached"Wait 30 seconds and refresh. If still failing, stop and start the system again.

When contacting the developer about an error, send the file jifs.log inside the jifs-fleet folder. It records everything the system does and speeds up diagnosis.
