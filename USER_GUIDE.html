Chapter 0

What Was Delivered

A summary of the complete system built for JIFS Company

The JIFS Fleet Management System is a custom-built software application that runs directly on your office computer. It does not need an internet connection. Staff can access it from any computer connected to the same office Wi-Fi network.

What the System Does

FeatureWhat it allows you to do
VehiclesRegister every truck in the fleet with plate number, type, model, and a monthly budget for tracking spending.
SuppliersStore details for fuel companies, garages, insurance providers, and other vendors you deal with.
ExpensesRecord every debit and credit transaction for each truck — fuel, maintenance, insurance, parts, and more. Auto-generates document numbers.
Print StatementGenerate a supplier ledger that matches the format of statements you receive, for easy reconciliation.
ServicesLog every service event and set next-service dates. The system alerts you when service is due.
PartsTrack installed spare parts with expiry dates. Alerts appear before parts need replacing.

Chapter 1

Installing & Starting the System

How to start the system, set up auto-start, and allow other computers to connect

⚠️
One-time requirement: Node.js must be installed

Node.js is a small program that runs the JIFS system. It should already be installed. If the system does not start, see the troubleshooting section at the end of this guide.

Starting the System (Every Day)

To start JIFS Fleet, simply double-click the file start-jifs.command in the jifs-fleet folder.

  1. 1
    Find the jifs-fleet folder on the Desktop or wherever it was saved on the office computer.
  2. 2
    Double-click "start-jifs.command". A black terminal window may flash briefly — this is normal. Wait a few seconds.
  3. 3
    The browser opens automatically and goes to the JIFS Fleet login page. A notification also appears in the top-right corner of the screen showing the system address.
  4. 4
    Log in with your username and password (see Chapter 2).
🖥️
What you will see
A macOS notification in the top-right corner that says "JIFS Fleet Started" and shows the LAN address (e.g. http://192.168.1.100:3000)

Stopping the System

When you are done for the day and want to shut the system down completely:

  1. 1
    Double-click stop-jifs.command in the jifs-fleet folder.
  2. 2
    A notification confirms the system has stopped. The browser will no longer be able to open the system.
💡
You do not need to stop it every day

It is fine to leave the system running overnight. Stop it only when the office computer will be switched off for an extended period, or when you are done for the week.

Auto-Start at Login (Recommended — One Time Only)

If you want the system to start automatically every time the computer is switched on — without needing to double-click anything — do this once:

  1. 1
    Double-click install-autostart.command in the jifs-fleet folder.
  2. 2
    A notification will confirm it is installed. From now on, JIFS Fleet starts by itself every time the Mac is switched on.
  3. 3
    Open the browser and go to http://localhost:3000 to access the system.
✅
After auto-start is installed, you only need to open the browser

The system starts in the background when the Mac turns on. Staff just open Chrome and go to http://localhost:3000.

Accessing from Other Computers on the Network

Any other computer connected to the same office Wi-Fi can access JIFS Fleet — no installation needed on those computers.

  1. 1
    Find the office computer's IP address.
    On the Mac that runs JIFS: click the Apple icon → System Settings → Wi-Fi → Details.
    Look for the "IP Address" — it will look like 192.168.1.100.
  2. 2
    On any other computer or tablet on the same Wi-Fi, open a web browser and type:
    http://192.168.1.100:3000 (replace with the actual IP address you found above)
  3. 3
    The JIFS Fleet login page will appear. Log in normally.
⚠️
The office computer (server) must be on

For other computers to access the system, the main office Mac must be switched on and JIFS Fleet must be running. If the main computer is off, no one can access the system.

Bookmarking the System in Chrome

To avoid typing the address every time, save it as a bookmark:

  1. 1
    Open the system in Chrome (go to http://localhost:3000).
  2. 2
    Press Ctrl + D (Windows) or ⌘ + D (Mac) to bookmark it.
  3. 3
    Name it "JIFS Fleet" and save. Next time, just click the bookmark.

Chapter 2

Logging In & User Accounts

How to sign in, what each user role can do, and your first-time admin setup

First-Time Login — Creating the Admin Account

When the system is started for the very first time, there are no user accounts yet. You need to create the main Admin account once using the following steps:

  1. 1
    Start the system (see Chapter 1) and open it in the browser.
  2. 2
    On the Login page, look for the link "Register" or "Create Account" below the login form.
  3. 3
    Enter a username (e.g. admin), a strong password (at least 8 characters with at least one number), and select Role: Admin.
  4. 4
    Click Create Account. You are now logged in as the system administrator.
  5. 5
    Write down the username and password and keep it in a safe place.
🚫
Keep the Admin password safe

The Admin account has full control over the system including deleting records and managing all users. Do not share it with everyone — create separate Operator accounts for daily staff (see Chapter 12).

Signing In

  1. 1
    Open the browser and go to http://localhost:3000.
  2. 2
    Type your username in the Username box.
  3. 3
    Type your password — it will appear as dots for security.
  4. 4
    Click "Sign In". The Dashboard appears immediately.

Session — How Long You Stay Logged In

Once you log in, your session stays active for 8 hours. You do not need to log in again if you close and reopen the browser within those 8 hours. After 8 hours of inactivity, the system will ask you to log in again for security.

Signing Out

Click the red Logout button at the very bottom of the left-hand menu. This is important in a shared office so no one else can use your account.

Understanding Debit vs Credit

📤
Debit = Money you OWE or SPENT

You bought something. Record the cost in Debit.
Example: Bought 300L diesel for 450,000 → Debit: 450000

📥
Credit = Money JIFS PAID back

JIFS paid a supplier. Enter the payment in Credit — it reduces what you owe.
Example: Paid supplier 200,000 → Credit: 200000

Copying an Expense (Fast Entry for Multiple Trucks)

When the same type of expense applies to several trucks (e.g. all trucks refuelled at the same station on one day), use the Copy button to avoid re-typing the form each time:

  1. 1
    Record the first truck's expense normally.
  2. 2
    Find that row in the list and click the grey Copy button.
  3. 3
    The form opens pre-filled. Change only the truck plate and amount, then click Create Expense.
  4. 4
    Repeat for each remaining truck.

Editing an Expense

Click Edit next to any entry. Change the fields you need and click Update Expense. The Document Number stays the same.

Filtering Expenses

Use the filter bar at the top of the Expenses page to find entries:

  • Search — type any word (plate, supplier name, description, reference number).
  • Type — show only Fuel, Maintenance, Insurance, Parts, or Other.
  • Payment Status — show only Unpaid, Partial, or Paid.
  • Date range — From and To dates to narrow to a specific period.

Click "Clear filters" to go back to the full list.

Print Statement — Supplier Reconciliation

Compare your records with the supplier's statement and confirm the balance matches

Every month, suppliers like Said Salim Bakhresa & Co. send JIFS a statement listing all transactions and the outstanding balance. The Print Statement page lets you generate JIFS's own version of this ledger so you can compare them side by side. This is called reconciliation.

Generating a Statement

  1. 1
    Click Print Statement in the left menu. The current month loads automatically.
  2. 2
    To change the date range: update the From Date and To Date fields to match the supplier's statement period.
  3. 3
    To filter by supplier: click one or more supplier name buttons (they turn blue when selected). Click "All Suppliers" to see everything combined.
  4. 4
    Click "Generate Statement". The table and totals refresh.

Reading the Statement Columns

Summary Totals

Below the table are three boxes: Total Debit (all amounts owed), Total Credit (all payments made), and Closing Balance (what is still outstanding). These should match the totals at the bottom of the supplier's own statement.

Chapter 11

Reports — Financial Analysis

8 different report views to understand where your money is going

The Reports page has 8 tabs. Click any tab across the top to switch between report types. Reports with a date filter show a From / To date bar at the top — set the period you want and the report updates automatically.

Tab 1 — By Vehicle

Shows total spending per truck within a selected period. Columns: number of transactions, Total Debit, Total Credit, and Net Cost (Debit minus Credit). Useful for comparing which trucks are costing the most.

Tab 2 — By Category

Shows how much has been spent in each category — Fuel, Maintenance, Insurance, Parts, Other — within the selected period. Helps you understand where the money is going.

Tab 3 — Transactions

A full line-by-line list of every expense within the selected date range. The footer row shows total debits and total credits for the period. You can print this as a full transaction report.

Tab 4 — Fuel

Fuel-specific statistics per truck for the selected period:

  • Fill-ups — how many times fuel was purchased for each truck
  • Total Litres — total fuel volume in litres
  • Total Cost (TZS) — total money spent on fuel
  • Cost per Litre — automatically calculated average price per litre

Tab 5 — Supplier Balances

Shows the outstanding balance owed to each supplier across all time (not filtered by date). The "Outstanding Balance" column shows total Debit minus total Credit. An amber or red amount means JIFS still owes that supplier money. Use this to prioritise payments.

💡
Use Supplier Balances + Print Statement together

Supplier Balances gives a quick overview of all suppliers. When you need the full detail for one supplier (to compare with their statement), switch to Print Statement.

Tab 6 — Services Schedule

Two sections: ● Overdue Services (service date already passed — shows how many days overdue) and ● Upcoming (next 60 days) (colour-coded: green = safe, amber = within 14 days, red = within 7 days).

Tab 7 — Parts Expiry

Two sections: Expired Parts (parts that have already passed their expiry date) and Expiring in next 90 days (colour-coded by urgency). Use this to plan purchasing of replacement parts.

Tab 8 — Monthly Overview

A complete breakdown for a single chosen month. Four summary cards compare spending against the previous month (green = went down, red = went up). Two tables show the breakdown by category and by truck for that month.

Printing Any Report

Once a report is loaded, click the blue Print button. Only the report table is printed — the sidebar and filter controls are hidden. You can also save as PDF from the print dialog.

Chapter 13

Daily & Monthly Workflow

A practical guide for running the system day-to-day

First-Time Setup (Do This Once)

Before recording any transactions, complete these steps in order:

Setup Order

  1. 1
    Create the Admin account — see Chapter 2. Write the password somewhere safe.
  2. 2
    Add all vehicles (Chapter 4) — every truck with its plate number. Set monthly budgets if desired.
  3. 3
    Add all suppliers (Chapter 5) — every company you buy fuel, parts, or services from.
  4. 4
    Add vehicle documents (Chapter 10) — TRA sticker, road licence, and insurance for every truck with their expiry dates.
  5. 5
    Create staff accounts (Chapter 12) — add Operator accounts for anyone who will be recording transactions daily.
  6. 6
    Install auto-start (Chapter 1) — so the system starts automatically every morning.
  7. 7
    Start recording expenses — you are ready.

Every Morning (5 Minutes)

Morning Check

  1. 1
    Open the browser, go to http://localhost:3000 and log in.
  2. 2
    Check the Dashboard — look at the Budget Warnings (any truck spending too much?), Upcoming Services (any truck due for service this week?), and Vehicle Documents (any expired or near-expiry?).
  3. 3
    If any red or amber alerts appear, arrange action (renew document, book service, etc.) before the trucks leave.

Recording Daily Expenses

Fuel Entry

  1. 1
    Collect fuel delivery notes or receipts from drivers.
  2. 2
    Go to Expenses → Add Expense.
  3. 3
    Select truck, Type = Fuel, select supplier, enter litres in Quantity (unit = Litres), enter cost in Debit, and enter the odometer reading.
  4. 4
    For multiple trucks at the same station: record the first one, then use Copy for each remaining truck and just change the plate and amount.

Recording a Payment to a Supplier (Credit)

When JIFS pays a supplier, record it as a new expense entry but put the payment amount in the Credit field (not Debit). Leave Debit at zero. Select the same supplier. This reduces the outstanding balance on the statement.

Also go back and update the Payment Status of the related invoices to Paid or Partial.

End of Month (30 Minutes)

Month-End Checklist

  1. 1
    Reconcile supplier statements: For each supplier that sent a statement, go to Print Statement, set the date range to match, select the supplier, and generate. Compare each line and the closing balance with the supplier's document.
  2. 2
    Review monthly spending: Go to Reports → Monthly Overview, select last month, and check whether spending went up or down vs. the previous month. Look at which trucks and categories changed the most.
  3. 3
    Check unpaid invoices: Go to Expenses, filter by Payment Status = Unpaid. Review what is outstanding and follow up on payments.
  4. 4
    Back up the database: Double-click backup-jifs.command to create a backup copy of all your data. See Chapter 14.

Chapter 14

Backup & Data Safety

How to protect your data so nothing is ever lost

🚫
Your data can be lost if the computer is damaged, stolen, or broken

All data is stored in a single file on the office computer. If that computer is lost or the hard drive fails, all records are gone — unless you have a backup. Back up regularly. This is the single most important maintenance task.

How to Make a Backup (Recommended: Weekly)

  1. 1
    Double-click backup-jifs.command in the jifs-fleet folder.
  2. 2
    A backup is automatically saved to a folder called JIFS-Backups on the Desktop. The file name includes the date and time (e.g. fleet_2026-04-22_09-30.db).
  3. 3
    A notification confirms the backup is complete and opens the JIFS-Backups folder.
  4. 4
    Copy the backup file to a USB drive or an external hard disk and store it somewhere separate from the computer.
💡
The system keeps the last 30 backups automatically

You do not need to manually delete old backup files. The system removes the oldest ones when you have more than 30, keeping your disk from filling up.

Where is the Database File?

If you ever need to locate the database file manually:

jifs-fleet / backend / data / fleet.db

This single file contains all your data — every vehicle, supplier, expense, service, part, and document. Copying this file is a complete backup.

Restoring from a Backup

If something goes wrong and you need to restore your data, contact your system developer (see contact details at the end of this guide). They will restore the backup file to the correct location. Do not attempt this yourself unless instructed.

Other Safety Tips

  • Do not delete, move, or rename the fleet.db file.
  • Do not delete the jifs-fleet folder or any files inside it.
  • Keep the office computer's operating system up to date (macOS updates).
  • Use a UPS (uninterruptible power supply) to prevent data corruption from sudden power cuts.
  • Back up to a USB drive stored in a different room or building from the computer.

Chapter 15

Troubleshooting & Common Questions

Answers to the most common problems users encounter

The system does not open in the browser

⚠️
The system must be started first

Before opening the browser, double-click start-jifs.command in the jifs-fleet folder. Wait about 10 seconds. Then open Chrome and go to http://localhost:3000.

If the system still does not open after starting it, check that Node.js is installed. Ask your developer or contact Seynation Digital LTD for assistance.

I cannot find my truck in the expense form

Only Active vehicles appear in the vehicle dropdown. Check that the truck is registered in Vehicles and its status is set to Active.

I made a mistake in an expense entry

Go to Expenses, find the entry, and click Edit. Correct the fields and click Update Expense. If the entry must be removed completely, an Admin can click Del — deleted records cannot be recovered.

The statement balance does not match the supplier's statement

  • Check that every invoice from the supplier for that period has been entered in Expenses.
  • Check that all payments made to the supplier are recorded as Credit entries.
  • Make sure the date range exactly matches the supplier's statement period.
  • Check for any entry where Debit and Credit amounts may have been accidentally swapped.

I cannot delete a vehicle or supplier

If you see a message saying it cannot be deleted, it has expense records linked to it. Do not delete — set the vehicle to Inactive instead, or leave the supplier as is. The data is protected intentionally.

A user forgot their password

An Admin must reset it. Go to Manage Users, click Edit next to that user, enter a new password in the Reset Password field, and click Save Changes. Tell the user their new password privately.

A user account is locked

Go to Manage Users, click Edit next to the locked account (shown with a red Locked badge), reset the password, and save. The lock is cleared immediately.

The browser shows the system but another computer cannot connect

  • Make sure the main office computer is turned on and JIFS Fleet is running.
  • Confirm both computers are on the same Wi-Fi network.
  • Re-check the IP address (System Settings → Wi-Fi → Details on the Mac).
  • Type it as http://192.168.x.x:3000 (replace with your actual IP).

The start-jifs.command file will not open

On a Mac, the first time you run a .command file it may ask for permission. Right-click the file, choose Open, and click Open again in the security dialog. After the first time, it will open normally with a double-click.
